# Functional Requirement Table

| ID        | Feature | Description | Acceptance Criteria |
|-----------|---------|-------------|---------------------|
| VCR-SA-001 | Generate Product Voucher (LLM Coin, Model Restriction) | The system must allow a superadmin to create and configure "Voucher Product" for LLM Coins, with the option to restrict it to specific LLM models. | 1. The superadmin must be able to create a voucher by selecting the "Voucher Product" type (formerly Voucher Trial).<br>2. The superadmin must define the number of LLM Coins for the voucher.<br>3. When creating the voucher, the superadmin must be able to select specific models from the available LLM list.<br>4. The selected LLM model binding on a voucher cannot be changed after the voucher is created.<br>5. An optional campaign name can be added. |
| VCR-SA-002 | Generate GPU Credit Voucher | The system must allow a superadmin to create and configure vouchers for GPU Credit. | 1. The superadmin must be able to create a voucher by selecting the "Voucher Credit" type.<br>2. The superadmin must define the credit amount in IDR.<br>3. An optional campaign name can be added to the voucher batch. |
| VCR-SA-003 | Voucher Code Generation and Quota (Multiple User, Single/Multi-Code) | The system must allow the superadmin to define the quantity of vouchers to be generated and specify the code format. | 1. A mandatory "Voucher Quota" field determines the total number of vouchers to be generated in a batch, on both GPU credit voucher and product voucher.<br>2. The superadmin must select a mandatory "Code Type," which will be a checkbox of "Multi Code Generation". If checked, it will generate a unique, random code for each voucher in the quota. If not checked, it will generate one reusable code for the entire quota. |
| VCR-SA-004 | Voucher Management | The system must provide superadmins with the ability to access, view, and organize the list of all created voucher batches. | 1. Access to the "Manage Voucher" menu must be restricted to the superadmin role only.<br>2. The list of vouchers created must display comprehensive details.<br>3. Superadmins must be able to sort and filter the voucher list based on type (GPU/LLM), status (active/expired), redemption status, and campaign name. |
| VCR-SA-005 | Voucher Monitoring | The system must allow superadmins to monitor the detailed usage and status of generated vouchers. | 1. A "Voucher Status" view must be available to see claim details: who redeemed the voucher (name & email), redemption status, and redemption timestamp.<br>2. A "Voucher History" view must be available to see historical data of used vouchers.<br>3. The interface must display how many vouchers have been used and how many remain for a given batch. |
| VCR-SA-006 | Voucher Data Download | The system must allow superadmins to download raw voucher codes and formatted activity reports. | 1. Superadmins must be able to download the list of generated voucher codes in CSV format. |
| VCR-UA-001 | Voucher Redemption | The system must allow a regular user to claim a voucher code and have the system validate it against all defined rules. | 1. The user enters a voucher code to claim it.<br>2. The system will validate that the voucher is still active.<br>3. The system will validate the redemption quota for the code has not been met.<br>4. The system will validate that the user has not previously claimed a voucher from this batch.<br>5. If it is an LLM voucher, the system will validate that it is being applied to the correct model as specified during creation. |
